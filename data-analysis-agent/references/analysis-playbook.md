# Analysis Playbook

## 1. 问题定义模板

- 业务问题: 要解释什么现象或支持什么决策
- 指标定义: 指标公式、分母分子、时间口径
- 分析范围: 时间区间、用户/商品/渠道范围
- 成功标准: 结论如何被业务采纳

## 2. 数据质量检查清单

- 完整性: 缺失率、关键字段非空率
- 一致性: 字段类型、单位、时区是否一致
- 唯一性: 主键冲突、重复记录比例
- 合理性: 数值是否越界、时间是否倒序

## 3. 常见分析路径

- 趋势分析: 按天/周/月观察指标变化，识别拐点
- 结构分析: 按渠道、地区、品类分解贡献度
- 漏斗分析: 各环节转化率与流失点定位
- 留存分析: 分 cohort 观察留存曲线与差异
- 实验分析: 对照组与实验组差异、显著性检验

## 4. 输出模板

1. 结论摘要（3 条以内）
2. 关键证据（图表与核心统计量）
3. 影响评估（规模、风险、边界）
4. 行动建议（优先级、负责人、时间）
5. 复现信息（SQL/脚本/参数）
